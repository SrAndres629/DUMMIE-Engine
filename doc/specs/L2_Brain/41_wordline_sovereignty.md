# Spec 41: 4D Wordline & 6D Context Sovereignty Protocol

## 1. El Concepto de Wordline (Línea de Mundo)
Toda acción del enjambre es un punto inmutable en una Wordline. El servidor SSH local es el único medio autorizado para la lectura de baja entropía de este continuo.

## 2. Redefinición de Dimensiones

### 2.1 El Grafo 4D (Estructura de la Verdad)
- **Wordline-T:** Orden cronológico-causal (Lamport Clock).
- **Locus-S:** Ubicación física en el árbol de archivos (mapeado vía SSH).
- **Entropy-E:** Nivel de deuda técnica o ambigüedad generada.
- **Topology-G:** Relaciones de dependencia entre capas del motor.

### 2.2 El Sextante 6D (Navegación Contextual)
Para recuperar información, el agente debe proporcionar las coordenadas:
`[X:Departamento, Y:Capa, Z:Fichero, T:Tiempo, A:Autoridad, I:Intento]`

## 3. Protocolo de Consciencia .aiwg
El sistema ejecutará un "Bucle de Reflexión" cada 10 turnos:
1. **Audit:** El `Context Manager` lee el ledger vía SSH.
2. **Synthesize:** El `Formal Validator` identifica ineficiencias.
3. **Upgrade:** El `Contract Architect` propone cambios en las reglas del sistema para optimizar el consumo de tokens y la precisión.

## 4. Estrategia de Ahorro vía SSH
- **Command Injection over Streaming:** Preferir `grep`/`awk` vía SSH sobre `read_file`.
- **Remote Diffing:** Realizar el diff de specs localmente y solo enviar el resumen de cambios al LLM.
